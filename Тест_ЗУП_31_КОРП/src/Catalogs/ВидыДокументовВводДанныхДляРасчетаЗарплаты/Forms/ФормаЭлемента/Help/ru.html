Шаблон документа ввода исходных данных для расчета зарплаты

На различных предприятиях используются различные системы оплаты труда и различные способы доведения до расчетных служб значений числовых показателей, которые влияют на расчет зарплаты.

Программа предоставляет возможность описать не только разнообразные начисления, рассчитываемые по произвольным формулам, но и описать виды документов, которые будут использоваться для ввода значений показателей, используемых в этих формулах.

Требуется лишь однажды описать вид такого документа, ориентируясь на потребности конкретного предприятия, и впоследствии значения показателей можно будет вводить с помощью документа наиболее удобного или привычного вида.


Настройка шаблона для ввода исходных данных позволяет определить вид и функциональность документов Данные для расчета зарплаты, с помощью которых по этому шаблону можно будет вводить:

• значения показателей расчета зарплаты

• данные об отработанном или неотработанном времени

Если по настроенному шаблону уже были введены документы, то изменение его настроек недоступно, если оно нарушает целостность ранее введенных документов.

Настройка шаблона

Наименование шаблона служит лишь для обозначения разных шаблонов в их списке.

Вводимые по шаблону документы будут представлены с названием, указанным в поле Представление документа, а в самих документах может отображаться подсказка, если ее заполнить.

То, какие данные предполагается вводить документами по настраиваемому шаблону, указывается на отдельных закладках. При этом имеется возможность настраивать шаблоны для ввода данных в различных комбинациях, например, одним документом регистрировать как значение показателей расчета зарплаты, так и данные о времени.

Имеется возможность разработать печатные формы и подключить их к шаблону. Тогда их можно будет распечатать из документов, создаваемых по этому шаблону. Подключение производится в отдельной форме, открываемой по одноименной кнопке.

Закладка "Показатели зарплаты"

Если документ предполагается использовать для ввода значений показателей расчета зарплаты, то следует заполнить закладку Показатели зарплаты.
Одним видом документа предполагается регистрировать либо разовые числовые и денежные показатели (любого назначения) либо постоянные для подразделения и/или организации. Значения постоянных показателей для сотрудников регистрируются кадровыми документами.

Флажками следует отметить показатели, значения которых предполагается вводить. Создать при необходимости новый показатель можно непосредственно отсюда по кнопке Добавить показатель.

Ввод единовременно вводимых разовых показателей впоследствии предполагается с указанием целого месяца, к которому они относятся. Например, процент премии за май.

Накапливаемые разовые показатели будут вводиться с указанием конкретной даты (дня), на которую учитывается вводимое значение. Например, количество заключенных договоров, заключенных 15 мая.

Постоянные показатели буду вводиться также с указанием конкретной даты (дня), с которой действует их значение. Например, процент премии по подразделению с 1 мая.

При использовании в шаблоне последних двух видов показателей имеется возможность установить флажок Значения показателей вводятся на разные даты. Тогда одним документом можно будет регистрировать значения:
• накапливаемых разовых показателей на несколько разных дат. Например, для последовательного заполнения одного документа в течение месяца;
• постоянных показателей, действующих с нескольких разных дат. Например, для установки значений для нескольких подразделений или организаций одним документом с разных дат.

При использовании постоянных показателей также имеется возможность установить флажок Использовать дату окончания. Это позволит указать дату, начиная с которой вводимое значение показателя перестанет использоваться.

Другие возможности настройки шаблона представлены на закладке Дополнительно.

Закладка "Виды времени"

В программе имеется возможность регистрировать фактическое время не только специально предусмотренным документом Табель учета рабочего времени, но и документом Данные для расчета зарплаты.
Эту возможность удобно использовать, например, если предполагается ввод данных о времени пользователем, ответственным за учет определенного вида времени. Например, старший по торговому залу вводит данные о времени работы сотрудников в зале и на кассе. Подробнее см. учет времени.

Для задействования этой возможности следует заполнить закладку Виды времени – отметить в ней те виды времени, которые предполагается вводить отдельным документом.

Тогда документ Данные для расчета зарплаты будет выглядеть аналогично табелю – появится возможность вводить на каждый день месяца часы по выбранным видам времени.
Если установить переключатель в разделе Время вводится в положение отдельными документами на разные даты, то в один документ можно будет ввести количество часов только в один выбранный день.

Имеется возможность сводного ввода времени. Для этого следует установить флажок В целом за период. В таком случае в документе можно будет вводить количество дней и часов в целом за выбранный месяц, без детализации по дням.

Закладка "Дополнительно"

Имеется возможность повысить удобства ввода документов, заранее предусмотрев наиболее подходящий способ их заполнения.
Доступность настроек заполнения зависит от того, показатели какого назначения будут вводиться в документе, также контролируется их допустимая взаимная зависимость.

Одним документом может вводиться значение как для одного объекта (сотрудника, подразделения или организации), так сразу для нескольких. В этом случае в документе будет доступно не поле для ввода одного значения, а таблица для ввода нескольких значений.

Можно настроить документ так, чтобы таблица сразу же автоматически заполнялась всеми объектами (сотрудниками, подразделениями или организациями). В этом случае значения нужно будет ввести для всех объектов. Если эта возможность не выбрана, в документе все же будет возможность автоматического заполнения по отдельной кнопке.

Также доступны настройки, которые позволяют определить способ выбора организации и подразделения, по которым будут вводиться значения.
