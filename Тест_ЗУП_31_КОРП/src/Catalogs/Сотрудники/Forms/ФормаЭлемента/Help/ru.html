Сотрудник

Под сотрудниками в программе понимаются как работники, с которыми оформлены трудовые отношения (трудовой договор), так и договорники – подрядчики и авторы, с которыми оформлены гражданско-правовые отношения (договор на выполнение работ или оказание услуг, договор авторского заказа).

Способы создания сотрудников в программе

Карточка сотрудника

Если в программе ведется учет по нескольким организациям, то в карточке сотрудника следует указать организацию, с которой оформляется трудовой договор. При этом если трудовой договор оформляется в обособленном подразделении, выделенном на отдельный баланс, то в карточке указывается головная организация этого подразделения. Само такое обособленное подразделение будет указываться в документе Прием на работу.
По умолчанию организация заполняется организацией, указанной текущим пользователем на его рабочем столе.

Основные личные данные

Фамилия, имя и отчество вносятся в поле ФИО одной строкой, однако программой они разбиваются на три строки (фамилия, имя и отчество) и хранятся отдельно. ФИО, состоящее более чем из трех слов, также вносится в это поле целиком.

Если человек изменил фамилию, имя или отчество, то это следует отразить в отдельном окне по ссылке Изменить ФИО. В этом случае история изменения ФИО будет сохранена в программе, что важно, например, при формировании отчетности. Непосредственно поле ФИО следует редактировать в том случае, если при вводе была допущена опечатка. Просмотреть и отредактировать историю изменения ФИО можно по ссылке История ФИО.

Чтобы не допустить дублирования физических лиц, их ввод автоматически контролируется. Если после ввода ФИО обнаруживается, что физическое лицо с такими же или похожими данными уже было зарегистрировано в программе, то программа сообщит об этом в отдельном окне.

Табельный номер формируется программой автоматически при записи нового сотрудника, как следующий по порядку за последним, который уже есть в программе. При необходимости его можно отредактировать вручную.
В настройках кадрового учета предусмотрена возможность отключения контроля уникальности табельных номеров.

Помимо ФИО, непосредственно в карточке можно указать другие основные личные данные: дату рождения, пол, ИНН и СНИЛС. Другие личные данные указываются в отдельной форме, которая открывается по одноименной ссылке.

Если в программе используется ограничение доступа на уровне записей с точностью до физических лиц, то следует указать группу доступа, к которой относится этот человек. Тогда он будет доступен только тем пользователям, права доступа которых предполагают работу с этой группой.

Текущее место работы и другие места работы

Для вновь внесенных в программу сотрудников доступны ссылки для оформления
документов приема на работу, а также договоров ГПХ, если при настойке программы выбрана возможность регистрировать выплаты по таким договорам.

Если на сотрудника в программе уже оформлен прием на работу по трудовому договору, то в карточке указывается дата приема и информация о текущем месте работы, такая как подразделение, должность, условия оплаты труда. Эта информация доступна только для просмотра, при необходимости ее изменить следует внести изменения в кадровые приказы. К приказам можно перейти по ссылке История изменения мест работы.
Дата приема доступна для редактирования только в том случае, если сотрудник оформлен при начале эксплуатации программы с помощью документа Данные на начало эксплуатации. В этом случае ее следует заполнить датой приема сотрудника на это рабочее место в действительности.
После оформления увольнения сотрудника в карточке показывается дата увольнения.

Если человек имеет на предприятии другие рабочие места, т.е. ему соответствует несколько сотрудников программы, то в карточке отображается информация о таких рабочих местах – по ссылке можно открыть карточку другого соответствующего ему сотрудника.

Дополнительные данные

Если с сотрудником в программе регистрируется несколько трудовых договоров, т.е. для одного человека создается несколько сотрудников, то для того чтобы быстро различать таких сотрудников в списках и отчетах программы, можно задать произвольную строку, которая будет дополнять представление (название) сотрудника. Например, для совместителя можно указать "(совм.)", а если сотрудник принимается повторно после увольнения, то для прошлого сотрудника можно указать "(увол.)".

"Устаревших" сотрудников дополнительно можно поместить "в архив", с помощью флага Не отображать в списках. Такие сотрудники не показываются в списках (однако при необходимости в списках можно указать, чтобы показались и они).

Имеется возможность самостоятельно добавить любые произвольные сведения о сотруднике и использовать их для вывода и списки и отчеты. Возможность выбирается в разделе Администрирование/Общие настройки путем установки флага Дополнительные реквизиты и сведения. После ее выбора становится доступно описание произвольных реквизитов и сведений для сотрудника. Заданные для сотрудника дополнительные реквизиты редактируются в этой карточке.

Другие разделы и возможности карточки

По ссылкам из карточки сотрудника можно открыть окна для просмотра и редактирования других сведений о нем:

  • Личные данные
  • Воинский учет (ссылка доступна, если в настройках кадрового учета выбрана возможность ведения воинского учета)
  • Образование, квалификация
  • Семья
  • Трудовая деятельность


Следующие разделы карточки доступны, после того как в программе уже зарегистрированы договорные отношения с сотрудником:

  • Страхование
  • Выплаты, учет затрат
  • Налог на доходы


Также для уже принятого сотрудника доступны разделы для просмотра оформленных на него документов и других сведений:

  • Договоры ГПХ (раздел доступен, если на сотрудника оформлен договор ГПХ)
  • Кадровые документы
  • Начисления и удержания
  • Отсутствия


В командной панели карточки (верхний ряд с кнопками) доступны меню:

  • Оформить документ – для удобства оформления различных документов для этого сотрудника;
  • Печать – можно сформировать приказы, справки, карточки и другие печатные формы.
