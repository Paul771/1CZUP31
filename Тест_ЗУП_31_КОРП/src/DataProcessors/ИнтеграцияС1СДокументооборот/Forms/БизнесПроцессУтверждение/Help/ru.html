Этот бизнес-процесс используется для передачи документа на утверждение руководителю. 

При старте бизнес-процесса руководителю поступает задача "Утвердить", содержащая ссылку на документ. После того как руководитель рассмотрит документ и отметит в задаче результат утверждения, автору поступает задача "Ознакомиться с результатами утверждения". В случае отрицательного результата из этой задачи автор может отправить документ на повторное утверждение. 

Этот бизнес-процесс можно создавать непосредственно из карточки документа при помощи меню "Создать бизнес-процесс".

При выборе утверждающего можно указать не только конкретного пользователя, но и роль.

Для просмотра текущего состояния бизнес-процесса (списка выполненных и не выполненных задач) используйте команду "Задачи" в панели навигации карточки документа, отправленного на утверждение.

Если не был выполнен запуск бизнес-процесса, заполнение формы может быть выполнено на основании шаблона бизнес-процесса. Для этого необходимо выполнить команду "Заполнить по шаблону" на командной панели формы. Если для данного типа бизнес-процесса и вида предмета бизнес-процесса настроены шаблоны бизнес-процессов, то система предложит выбрать один из шаблонов для заполнения формы. Если был настроен только один шаблон, то заполнение формы произойдет автоматически.

Если в системе Документооборот включена настройка программы "Использовать электронно-цифровые подписи и шифрование", становится доступной галочка "Подписывать ЭЦП". Если галочка "Подписывать ЭЦП" включена, при утверждении предмета ответственным лицом нужно будет установить электронно-цифровую подпись (ЭЦП) на предмет.
