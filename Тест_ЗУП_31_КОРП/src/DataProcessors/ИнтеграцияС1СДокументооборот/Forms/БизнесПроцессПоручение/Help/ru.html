Этот бизнес-процесс следует использовать для выдачи простых поручений сотрудникам.

После выполнения задачи исполнителем, она поступает на проверку проверяющему (по умолчанию - автор бизнес-процесса). Проверяющий имеет возможность вернуть исполнителю задачу на доработку. Пользователь, указанный в поле "Контролер", получит задачу одновременно с исполнителем. В обязанности Контролера входит слежение за сроками выполнения поручения.

Бизнес-процесс "Поручение" считается завершенным, после того как исполнитель выполнил свою задачу и проверяющий подтвердил выполнение, задача контролера при этом завершается автоматически.

Этот бизнес-процесс можно создавать непосредственно из карточки документа при помощи меню "Создать бизнес-процесс".

Для просмотра текущего состояния бизнес-процесса (списка выполненных и не выполненных задач) используйте команду "Задачи" в панели навигации карточки документа, по которому были выданы поручения.

Для того, чтобы направить задачу руководителю, рекомендуется использовать бизнес-процесс Рассмотрение.

Если не был выполнен запуск бизнес-процесса, заполнение формы может быть выполнено на основании шаблона бизнес-процесса. Для этого необходимо выполнить команду "Заполнить по шаблону" на командной панели формы. Если для данного типа бизнес-процесса и вида предмета бизнес-процесса настроены шаблоны бизнес-процессов, то система предложит выбрать один из шаблонов для заполнения формы. Если был настроен только один шаблон, то заполнение формы произойдет автоматически.
