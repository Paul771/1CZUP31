Задачи являются средством коммуникации между пользователями, а также используются для автоматизации процессов.

Для создания новой задачи можно воспользоваться одним из процессов. Для создания задач на основании других объектов программы используйте подменю "Создать на основании". При этом объект-основание станет предметом задачи.

Если по задаче назначено несколько исполнителей, то для ответственного исполнителя карточка задачи содержит список состояний выполнения работ соисполнителей для контроля.

С помощью команды Все задачи в карточке задачи можно посмотреть список всех других задач по данному процессу.

С помощью команды Принять к исполнению можно отметить задачу как принятую Вами. Принятая задача отображается обычным, а не жирным шрифтом. Если задача имела ролевую адресацию (т.е. была назначена не лично Вам, а роли, исполнителем которой Вы являетесь), то при выполнении команды Принять к исполнению адресация задачи будет изменена и она будет адресована лично Вам, а не роли. Это удобно использовать для "захвата" задач в том случае, если исполнителями роли являются сразу несколько сотрудников.
